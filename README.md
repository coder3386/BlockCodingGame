# BlockCodingGame
캡스톤 디자인 1 블럭코딩게임 개발용


요구사항

-오브젝트 : 플레이어 캐릭터, 도착지
-맵

- 플레이어 캐릭터가 있어야 한다.
- 플레이어 캐릭터는 상(점프) 좌 우 기능을 수행 할수 있어야 한다.
- 플레이어 캐릭터의 목표는 함정을 피해 도착지에 도착할수 있어야 한다.

- 플레이어 캐릭터의 움직임을 유저가 직접 만들수 있어야 한다.
- 플레이어 캐릭터의 움직임을 할수 있게 하는 코드 블럭이 존재해야한다.

- 코드블럭은 반복문 조건문의 기능을 할수 있어야한다.
- 코드블럭은 캐릭터의 이동을 조작할수 있어야한다.


- 코드블럭 사용방법을 알려주는 튜토리얼이 존재해야한다.
	- 튜토리얼 4스테이지
	- 1: 오른쪽 이동과 if문 블럭 주어져야한다.
	- 2: 점프 블럭이 주어져야한다.
	- 3: while문이 주어져야한다.
	---- 오른쪽 이동, 제자리 점프, 오른쪽 대각선 점프
 	- 4: 주어진걸 활용해서 나머지를 만들어야한다.(왼쪽 이동, 왼쪽 대각선 점프)

	-게임 플레이 1스테이지 만들기

- 메인화면이 있어야한다.
- 메인화면에는 플레이, 종료가 있어야한다.
- 플레이를 들어가면 스테이지 선택화면이 나와야한다.
- 진행상황을 저장할수 있어야한다.
- 게임 시작 전에 블럭코딩을 할수 있어야한다. + 맵 사진을 볼수 있어야한다.



---------------------------------------------------

- 적대적 몹이 존재해야 한다.
- 캐릭터는 공격 기능이 있어야 한다.
- 코드블럭은 유저가 직접 얻어야한다.
	----- 게임 진행중에 for문,case을 얻을수 있어야한다.
